# 2026-04-14

- Fixed `tdf-hq-ui` login cold-start handling: the backend boot app returns `503` plus `Retry-After: 5`, but the UI had been retrying auth after only `1s` and `2s` before surfacing a failure. Updated `src/api/auth.ts` to honor `Retry-After` and extended retries, and updated `src/pages/LoginPage.tsx` to poll `/health`, warm the backend, and keep login actions disabled while the service is still starting.
- Diagnosed the follow-up live outage on `tdf-hq.fly.dev`: the backend was not merely cold-starting, it was crash-looping during startup migrations with `SqlError ... cannot execute CREATE EXTENSION in a read-only transaction`. Patched `tdf-hq/src/TDF/App/Boot.hs` so startup only attempts `CREATE EXTENSION` when `pgcrypto` or `vector` are actually missing, which avoids killing boot on already-initialized read-only transactions.
