# 08 canal seguro privado

Usar esta carpeta solo para preparar el orden de documentos privados que se enviaran cuando un banco o cooperativa confirme un canal seguro. No hacer commit de cedulas, RUC reales, estados bancarios, declaraciones, escrituras, avaluos, contratos firmados, numeros de cuenta ni datos personales de clientes.

Archivos de trabajo seguros para git:

- `private-submission-index.md`: indice de la carpeta privada que se debe armar fuera de git.
- `secure-upload-manifest-template.csv`: plantilla para registrar documentos enviados por portal, correo institucional seguro o asesor identificado.
- `signed-proforma-replacement-checklist.md`: lista de verificacion para convertir presupuestos de planificacion en proformas firmadas.
- `private-submission-index.pdf`: version PDF del indice privado.
- `signed-proforma-replacement-checklist.pdf`: version PDF de la lista de proformas.

Regla operativa:

1. Confirmar asesor, institucion, correo/portal y producto antes de enviar documentos privados.
2. Subir el paquete privado solo al canal confirmado.
3. Registrar cada documento en el manifiesto.
4. Guardar copias reales en una boveda cifrada o drive privado, no en este repositorio.

La subcarpeta `documentos-reales/` existe solo como punto local de armado y esta ignorada por git. No colocar ahi nada que deba compartirse por commit; usarla solo como area temporal privada de esta maquina.
